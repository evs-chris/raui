A two- or more paned splitter that can be rendered as either vertical or horizontal. The split tries to be mobile friendly by having a large enough handle to hit with a finger and watching for touch events. Panes also come with minimize and maximize buttons by default that will shrink the pane to 0% and let the surrounding panes absorb its space. Split children are set up in their own panes, and you can use the special `pane` child to embed non-element content directly into a pane, as the contents of the `pane` element are used rather than the `pane` element itself. ### Plugin options All options are optional. * `name: string = 'split'` - the name to use when registering the plugin as a component ### Attributes * `vertical: boolean = false` - whether this split should have vertical panes. This defaults to `false`, so the default split uses horizontal panes. * `draggable: boolean = true` - whether the panes should be resizable by dragging the split handle * `maximizable: boolean = true` - whether the panes are maximizable * `minimizable: boolean = true` - whether the panes are minimizable #### Child attributes * `size: number` - the width of the pane as a percentage. This defaults to an equal amount of the space available in the split. * `minimize: boolean = false` - whether the split should start out minimized ### API * `maximize(index: number)` - maximize the pane at the given index * `minimize(index: number)` - minimize the pane at the given index
